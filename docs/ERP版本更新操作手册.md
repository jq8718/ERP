# ERP 版本更新操作手册

适用场景：服务器上已经安装过 ERP，现在要在上一版本基础上更新到最新版本。

## 本次更新内容

本次更新主要包含：

- 基础资料手工录入校验增强：
  - 物料最低库存、最近采购价不能填负数。
  - 客户产品必须选择生产成品，默认售价不能填负数。
  - 单位换算比例必须大于 0，源单位和目标单位不能相同。
  - 供应商采购价不能填负数，失效日期不能早于生效日期。
  - 产品组成清单失效日期不能早于生效日期。
- 产品组成清单成品下拉框只显示启用状态的成品物料。
- 修正财务对账测试中的固定日期问题，避免跨天后测试误报。
- 更新《ERP 各岗位人员详细操作手册》和 PDF 版本，新增录入校验说明和试运行检查表。

## 1. 更新前确认

更新前请确认：

- 当前 ERP 可以正常登录。
- 服务器上 PostgreSQL、Python、NSSM 已经安装过。
- 当前 ERP 安装目录通常是 `D:\ERP\app`。
- 当前 ERP 数据目录通常是 `D:\ERP\data`。
- 当前 ERP 备份目录通常是 `D:\ERP\backups`。
- 当前 ERP 日志目录通常是 `D:\ERP\logs`。

更新程序会保留：

- `.env` 配置文件。
- PostgreSQL 数据库。
- 附件目录。
- 备份目录。
- 日志目录。
- Python 虚拟环境 `.venv`。

更新程序会更新：

- ERP 程序代码。
- 页面模板。
- 数据库迁移。
- 静态文件。
- Windows 服务配置。

## 2. 准备新版安装包

在开发电脑上准备最新的 `ERP更新包_日期_时间.zip`。

这个 zip 解压后会得到一个完整的 `ERP安装包` 文件夹。

复制到服务器时，可以放到任意临时目录，例如：

```text
D:\ERP更新包\
```

注意：

- 不要只复制 `ERP-Update.exe`。
- 必须复制整个更新 zip，或解压后的整个 `ERP安装包` 文件夹。
- 如果复制的是 `ERP更新包_日期_时间.zip`，请先右键解压，再进入解压后的 `ERP安装包` 文件夹运行更新。
- 不要把更新包解压到 `D:\ERP\app` 里面，建议放在 `D:\ERP更新包\` 或 U 盘临时目录。

## 3. 执行更新

推荐做法：

1. 在服务器上打开新版 `ERP安装包` 文件夹。
2. 右键 `ERP-Update.exe`。
3. 选择“以管理员身份运行”。
4. 如果 Windows 弹出权限确认，点击“是”。
5. 程序提示安装目录时，默认是：

```text
D:\ERP\app
```

如果现场 ERP 安装在其他目录，请填写实际目录。

6. 程序显示更新源目录和目标目录后，输入：

```text
UPDATE
```

7. 等待更新完成。

更新成功后，窗口会显示：

```text
ERP update completed.
```

## 4. 如果双击闪退

请使用控制台版本：

1. 右键 `ERP-Update-Console.cmd`。
2. 选择“以管理员身份运行”。
3. 按窗口提示操作。
4. 如果失败，查看日志目录：

```text
ERP安装包\installer\logs
```

把最新的 `erp-update-*.log` 发给开发人员。

## 5. 更新后检查

更新完成后，请检查：

1. 浏览器打开：

```text
http://服务器IP:8000/
```

2. 使用管理员账号登录。
3. 打开“系统健康检查”，确认没有红色错误。
4. 打开“销售订单”“库存汇总”“经营看板”等常用页面。
5. 使用销售、仓管、财务账号各登录一次，确认菜单正常。
6. 打开“基础资料 -> 物料”，随机新增一条测试物料，确认负数库存/负数采购价会被系统拒绝。
7. 打开“基础资料 -> 客户 -> 客户产品”，确认未选择生产成品时不能保存。
8. 打开“基础资料 -> 物料详情 -> 新增供应商价格”，确认失效日期早于生效日期时不能保存。
9. 打开新版《ERP 各岗位人员详细操作手册.pdf》，确认可以正常阅读中文。

## 6. 回退说明

更新程序会在下面目录自动备份更新前的应用代码：

```text
D:\ERP\update-backups\
```

备份名称类似：

```text
erp-app-before-update-20260704-153000
```

如果更新失败，请不要手工删除数据库或附件。保留错误窗口和日志，联系开发人员处理。
