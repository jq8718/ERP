# 各岗位账号试运行操作手册

适用场景：ERP 已安装到 Windows 内网服务器，管理员已经可以登录系统。本文用于上线前让销售、采购、仓管、生产、财务、老板、管理员分别登录试运行，确认每个岗位只看到自己该看的界面，并能完成本岗位常用操作。

## 1. 试运行前准备

### 1.1 准备测试账号

建议先创建以下测试账号，密码先用临时密码，试运行结束后再改为正式密码。

| 岗位 | 建议用户名 | 用途 |
| --- | --- | --- |
| 销售 | sales01 | 测销售订单、送货单、客户资料 |
| 采购 | purchase01 | 测采购需求、采购单、进货单、供应商退货 |
| 仓管 | warehouse01 | 测库存、期初库存、库位、盘点、移库 |
| 生产 | production01 | 测生产指令、生产领料、生产入库 |
| 财务 | finance01 | 测收款、付款、期初应收应付、管理费用、经营看板 |
| 老板 | boss01 | 测只读查看经营数据和关键报表 |
| 管理员 | admin01 | 测用户、角色、权限、备份、健康检查 |

### 1.2 建议角色和权限

管理员进入“角色管理”，按下面建议创建角色。权限名称以系统页面显示的中文为准。

| 角色 | 必选权限 | 可选权限 | 说明 |
| --- | --- | --- | --- |
| 销售 | 查看本人销售数据、处理销售单据 | 查看客户和供应商联系信息 | 普通销售默认只看自己负责客户或自己创建的数据 |
| 销售主管 | 查看本人销售数据、查看全部销售数据、处理销售单据 | 查看财务金额、查看客户和供应商联系信息 | 主管可查看全部销售数据；是否看金额由公司决定 |
| 采购 | 查看采购数据、处理采购单据 | 查看客户和供应商联系信息、查看财务金额 | 不给“查看财务金额”时采购价格会脱敏 |
| 仓管 | 查看库存数据、处理库存单据 | 查看采购数据、查看生产数据 | 仓管主要看库存；如需看来源单据，可加采购/生产查看权限 |
| 生产 | 查看生产数据、处理生产单据 | 查看库存数据、查看 BOM | 生产需要查看产品组成和库存时，加对应查看权限 |
| 财务 | 查看财务金额、处理收付款和余额 | 查看本人销售数据、查看采购数据 | 财务默认可以看到金额、收付款和期初应收应付 |
| 老板 | 查看财务金额、查看全部销售数据、查看采购数据、查看库存数据、查看生产数据、查看 BOM | 查看客户和供应商联系信息 | 老板建议只读，不给处理类权限 |
| 管理员 | 权限与审批规则管理 | 删除附件、查看敏感附件 | 管理员负责账号、权限、健康检查和运维入口 |

注意：

- “查看”权限决定能不能进入模块和看列表。
- “处理”权限决定能不能新增、编辑、确认、作废、导入等。
- “查看财务金额”决定金额字段是否显示，没权限时会显示 `******`。
- “查看全部销售数据”决定销售是否能看到别人的客户和订单。

### 1.3 准备少量测试数据

试运行不用一开始导入全部真实数据，建议先准备少量样本：

| 数据 | 建议数量 |
| --- | --- |
| 客户 | 2 个 |
| 供应商 | 2 个 |
| 物料/成品 | 5-10 个 |
| 产品组成清单 | 1-2 个 |
| 库位 | 2 个 |
| 期初库存 | 每个关键物料 1 批 |
| 期初应收/期初应付 | 各 1 条 |
| 销售订单 | 1-2 张 |
| 采购单/进货单 | 1-2 张 |

## 2. 通用试运行规则

每个岗位都按以下顺序检查：

1. 用该岗位账号登录。
2. 看左侧菜单，只应该出现本岗位相关模块。
3. 打开本岗位常用列表页，确认能正常显示。
4. 尝试访问不属于本岗位的页面，应该被拒绝或菜单中看不到。
5. 执行一条本岗位常用操作。
6. 检查操作成功提示、单据状态、库存/金额变化是否合理。
7. 退出登录，换下一个岗位账号。

建议记录试运行问题：

| --- | --- | --- | --- | --- | --- | --- |

## 3. 销售账号试运行

### 3.1 登录后应看到

销售账号通常应看到：

- 工作台
- 系统消息
- 审批
- 销售订单
- 欠料提醒
- 销售出库
- 借样
- 借样归还
- 客户退货
- 客户
- 附件

普通销售不应看到：

- 财务经营看板
- 客户收款、供应商付款
- 管理费用
- 用户管理、角色管理、权限清单
- 系统健康检查

### 3.2 操作步骤

1. 进入“客户”，新建或查看一个客户。
2. 进入“销售订单”，新建销售订单。
3. 在销售订单中选择客户、填写交期、客户合同编号、结算方式和备注。
4. 在明细区域新增 1-2 行产品，测试“新增行”“删除行”是否正常。
5. 保存草稿。
6. 打开销售订单详情，提交审核或确认。
7. 如果库存不足，查看是否生成“欠料提醒”。
8. 如果库存满足，继续生成或查看“销售出库”。
9. 打开送货单打印页，确认不显示价格，并能看到公司抬头、客户合同号、地址、联系人、电话、结算方式和备注。

### 3.3 通过标准

- 普通销售只能看到自己负责客户或自己创建的销售数据。
- 不该看到的财务金额不显示，或显示为 `******`。
- 销售订单明细可以增减行。
- 送货单不带价格。
- 欠料时能看到欠料提醒。

## 4. 采购账号试运行

### 4.1 登录后应看到

采购账号通常应看到：

- 工作台
- 系统消息
- 审批
- 欠料提醒
- 采购需求
- 采购单
- 进货单
- 供应商退货
- 物料
- 供应商
- 附件

采购账号通常不应看到：

- 客户收款、供应商付款
- 管理费用
- 用户管理、角色管理

### 4.2 操作步骤

1. 进入“欠料提醒”，查看销售订单产生的欠料。
2. 从欠料生成或查看采购需求。
3. 进入“采购单”，新建采购单或从采购需求生成采购单。
4. 填写供应商、物料、数量、预计到货日期。
5. 进入“进货单”，根据采购单做进货入库。
6. 测试合格数量、不合格数量填写。
7. 如有不良品，进入“供应商退货”创建退货单。

### 4.3 通过标准

- 采购能处理采购需求、采购单、进货单。
- 进货确认后库存批次和库存汇总增加。
- 没有财务金额权限时，采购价格或金额应脱敏。
- 采购不能进入用户权限管理页面。

## 5. 仓管账号试运行

### 5.1 登录后应看到

仓管账号通常应看到：

- 工作台
- 系统消息
- 审批
- 库存汇总
- 库位
- 库存批次
- 库存流水
- 库位移库
- 盘点
- 物料
- 附件

仓管账号通常不应看到：

- 客户收款、供应商付款
- 管理费用
- 用户管理、角色管理

### 5.2 操作步骤

1. 进入“库位”，确认默认库位已建立。
2. 进入“库存汇总”，查看物料库存。
3. 进入“库存批次”，确认批次、库位、库存类型正确。
4. 进入“库存流水”，查看期初、采购入库、销售出库等流水。
5. 进入“手工期初建账”，录入一条测试期初库存。
6. 打开初始化任务，确认期初库存入账。
7. 进入“库位移库”，测试从一个库位移到另一个库位。
8. 进入“盘点”，创建盘点单并确认差异处理。

### 5.3 通过标准

- 期初库存手工录入后能生成待确认任务。
- 期初库存确认后，库存汇总、库存批次、库存流水一致。
- 移库后批次库位变化正确，总库存不变。
- 仓管不能看到财务收付款页面。

## 6. 生产账号试运行

### 6.1 登录后应看到

生产账号通常应看到：

- 工作台
- 系统消息
- 审批
- 生产指令
- 生产领料
- 生产入库
- 物料
- 产品组成清单
- 库存汇总
- 附件

### 6.2 操作步骤

1. 进入“产品组成清单”，查看成品由哪些原料/配件组成。
2. 进入“生产指令”，新建或查看一张生产指令。
3. 确认生产指令中成品、数量、计划日期、锁定清单正确。
4. 从生产指令生成“生产领料”。
5. 打开生产领料单，确认系统按产品组成清单带出建议领料明细。
6. 确认生产领料，检查库存扣减。
7. 进入“生产入库”，登记生产完成入库。
8. 确认生产入库，检查成品库存增加。

### 6.3 通过标准

- 生产能查看产品组成清单。
- 生产领料按清单自动计算建议用量。
- 领料确认后原料库存减少。
- 生产入库确认后成品库存增加。

## 7. 财务账号试运行

### 7.1 登录后应看到

财务账号通常应看到：

- 工作台
- 系统消息
- 审批
- 经营看板
- 客户收款
- 供应商付款
- 期初应收
- 期初应付
- 客户余额
- 供应商余额
- 对账单
- 管理费用
- 客户
- 供应商
- 附件

财务账号通常不应看到：

- 用户管理、角色管理、权限清单
- 系统健康检查

### 7.2 操作步骤

1. 进入“期初应收”，新建一条客户期初应收。
2. 进入“客户收款”，新建收款单。
3. 打开收款单详情，选择销售订单或期初应收进行核销。
4. 确认收款，检查销售订单或期初应收余额减少。
5. 进入“期初应付”，新建一条供应商期初应付。
6. 进入“供应商付款”，新建付款单。
7. 打开付款单详情，选择进货单或期初应付进行核销。
8. 确认付款，检查进货单或期初应付余额减少。
9. 进入“管理费用”，登记一条费用，例如运费、电费、房租或餐费。
10. 确认管理费用。
11. 进入“经营看板”，查看本月/本年销售、采购、收款、付款、费用和估算经营额。

### 7.3 通过标准

- 财务金额正常显示，不是 `******`。
- 期初应收/应付可新建、查看、核销。
- 收款/付款确认后余额正确变化。
- 管理费用确认后，经营看板费用金额变化。
- 财务不能进入权限管理和系统健康检查。

## 8. 老板账号试运行

### 8.1 登录后应看到

老板账号建议只读，通常应看到：

- 工作台
- 系统消息
- 审批
- 经营看板
- 销售订单
- 采购单
- 库存汇总
- 生产指令
- 产品组成清单
- 客户
- 供应商

老板账号通常不应看到：

- 新增、确认、作废等处理按钮
- 用户管理、角色管理、权限清单

### 8.2 操作步骤

1. 进入“经营看板”，查看本月和本年数据。
2. 进入“销售订单”，确认可以查看订单列表和详情。
3. 进入“采购单”，确认可以查看采购情况。
4. 进入“库存汇总”，查看关键物料库存。
5. 进入“生产指令”，查看生产进度。
6. 尝试执行新增或确认动作，系统应不允许。

### 8.3 通过标准

- 老板能看到经营情况和关键业务进度。
- 老板默认不处理单据。
- 不显示或不能使用高风险操作按钮。

## 9. 管理员账号试运行

### 9.1 登录后应看到

管理员账号应看到：

- 用户管理
- 角色管理
- 权限清单
- 审批规则
- 附件访问日志
- 导入任务
- 初始化任务
- 导出日志
- 打印日志
- 登录会话
- 备份记录
- 后台任务
- 发布记录
- 审计日志
- 系统健康检查

### 9.2 操作步骤

1. 进入“用户管理”，新建一个测试用户。
2. 进入“角色管理”，创建或修改一个角色。
3. 给测试用户分配角色。
4. 用测试用户登录，确认菜单符合权限。
5. 回到管理员账号，进入“登录会话”，强制下线测试用户。
6. 进入“系统健康检查”，查看数据库、附件目录、备份目录、后台任务、发布门禁状态。
7. 进入“备份记录”，确认最近备份存在。
8. 进入“审计日志”，确认刚才的用户/角色操作有记录。

### 9.3 通过标准

- 管理员能维护用户、角色、权限。
- 关键管理动作要求输入当前登录密码和操作原因。
- 强制下线用户后，被下线用户再次操作会要求重新登录。
- 健康检查无红色错误。

## 10. 一条完整业务试运行建议

建议上线前至少跑通下面一条完整流程：

1. 管理员创建销售、采购、仓管、生产、财务、老板账号。
2. 仓管建立库位和少量期初库存。
3. 生产或管理员维护产品组成清单。
4. 销售创建销售订单并确认。
5. 系统库存不足时生成欠料提醒。
6. 采购根据欠料生成采购需求和采购单。
7. 采购确认进货入库。
8. 仓管检查库存增加。
9. 生产创建生产指令、领料、入库。
10. 销售创建或确认销售出库，打印送货单。
11. 财务登记客户收款并核销。
12. 财务登记管理费用。
13. 老板查看经营看板。
14. 管理员检查审计日志、打印日志、备份和健康检查。

## 11. 试运行通过标准

全部满足以下条件，才建议进入正式使用：

- 每个岗位账号只能看到自己应该看到的菜单。
- 每个岗位可以完成自己的日常操作。
- 无权限页面不能通过直接输入网址访问。
- 金额字段按权限显示或脱敏。
- 销售订单、采购入库、生产领料、生产入库、销售出库、收付款能跑通。
- 库存汇总、库存批次和库存流水一致。
- 期初库存、期初应收、期初应付可以被业务负责人核对。
- 送货单打印格式已用真实纸张确认。
- 系统健康检查无阻塞错误。
- 至少完成一次备份，并确认备份文件存在。
