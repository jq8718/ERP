# ERP 各岗位人员详细操作手册

适用对象：管理员、销售、采购、仓管、生产、财务、老板，以及负责基础资料和产品组成清单维护的人员。

适用范围：Windows 内网部署后的日常使用。本文按“菜单位置、填写内容、操作步骤、检查结果、常见问题”来写，便于新员工照着操作。

## 1. 使用前统一说明

### 1.1 登录和退出

1. 打开浏览器。
2. 输入 ERP 地址，例如：

```text
http://192.168.1.16:8000/
```

3. 输入用户名和密码。
4. 登录后先看左侧菜单，只会显示当前账号有权限的模块。
5. 操作结束后，点击右上角“退出”。

### 1.2 修改密码

1. 登录 ERP。
2. 点击右上角“修改密码”。
3. 输入旧密码。
4. 输入新密码并确认。
5. 保存。

建议：

- 第一次登录后必须改密码。
- 不要多人共用一个账号。
- 财务、管理员、老板账号密码要单独保管。

### 1.3 权限和菜单的关系

系统里常见权限分为三类：

| 类型 | 含义 | 例子 |
| --- | --- | --- |
| 查看权限 | 能进入模块、看列表和详情 | 查看库存数据、查看采购数据 |
| 处理权限 | 能新增、编辑、确认、作废、导入 | 处理销售单据、处理库存单据 |
| 字段权限 | 能看到敏感字段 | 查看财务金额、查看客户和供应商联系信息 |

没有金额权限时，金额会显示为：

```text
******
```

这是正常的，不是系统错误。

### 1.4 二次验证和操作原因

高风险操作通常需要填写：

| 字段 | 怎么填 |
| --- | --- |
| 当前密码 | 当前登录账号的密码 |
| 操作原因 | 简短写清楚原因，例如“确认入库”“客户要求作废”“首次建立产品组成清单” |

这些内容会写入审计日志，方便以后追溯。

### 1.5 日期怎么填写

系统里的订单日期、交期、入库日期、收款日期、付款日期、BOM 生效日期等，通常都会显示日历选择框。

不同 Windows 版本、浏览器和区域设置下，日历显示样式可能不完全一样，这是正常现象。人工输入后，系统会在离开输入框或提交表单前，自动整理成下面这种统一格式，再写入数据库：

```text
YYYY-MM-DD
```

例如：

```text
2026-07-04
```

如果电脑没有弹出日历，也可以手工输入下面任意一种常见格式，系统会自动识别：

```text
2026-07-04
2026/07/04
2026.07.04
2026年07月04日
```

建议优先点击日历选择日期；手工输入时，年份写 4 位，月份和日期不要写错。数据库里保存的是日期值，不保存 Windows 本地显示格式，所以不同电脑填写不会导致日期格式混乱。

### 1.6 手工录入数据的校验规则

系统会在保存时检查手工填写的数据。看到红色错误提示时，说明数据没有保存成功，需要按提示改正后再保存。

常见规则：

| 类型 | 规则 | 例子 |
| --- | --- | --- |
| 必填项 | 带业务含义的关键字段不能为空 | 物料编码、客户名称、生产成品、数量、金额 |
| 数量 | 大多数业务数量必须大于 0 | 订单数量、采购数量、入库数量、生产数量 |
| 金额/价格 | 成本、售价、采购价、费用不能填负数 | `-1`、`-0.01` 会被拒绝 |
| 日期 | 结束日期不能早于开始日期 | BOM 失效日期不能早于生效日期 |
| 下拉选项 | 尽量从下拉框选择，不要自己乱填 | 结算方式、付款方式、供应商类型 |
| 明细行 | 单据通常至少要有一条有效明细 | 销售订单、采购单、退货单 |
| 库存批次 | 批次、物料、库位必须匹配 | 出库批次必须属于当前出库物料 |

如果保存失败：

1. 先看页面顶部或字段旁边的红色提示。
2. 不要刷新页面，先把错误字段改正确。
3. 数量和金额不要填负数。
4. 日期优先用日历选择。
5. 如果提示“至少需要一条明细”，检查是否只保存了空白行。

### 1.7 岗位和日常工作总览

不同岗位登录后看到的菜单可能不一样，这是权限控制的结果。试运行时建议按下面分工使用，不要多人共用一个账号。

| 岗位 | 主要负责 | 常用菜单 | 不建议操作 |
| --- | --- | --- | --- |
| 基础资料维护 | 建物料、客户、供应商、库位、单位换算 | 基础资料 | 财务收付款、库存调整 |
| 产品和 BOM 维护 | 建成品、建产品组成清单、启用默认 BOM | 基础资料、产品组成清单 | 销售收款、供应商付款 |
| 销售 | 客户订单、送货单、借样、客户退货 | 销售 | 直接改库存、直接改采购单 |
| 采购 | 欠料跟进、采购单、进货、供应商退货 | 采购与生产 | 客户收款、销售价格调整 |
| 仓管 | 期初库存、入库确认、出库、移库、盘点 | 库存与财务、采购与生产 | 修改订单金额、财务核销 |
| 生产 | 生产指令、领料、生产入库 | 采购与生产 | 收付款、客户资料维护 |
| 财务 | 期初应收应付、收款、付款、费用、经营看板 | 库存与财务 | 修改 BOM、直接改库存数量 |
| 老板或管理层 | 看经营数据、看异常、看进度 | 看板、报表、各模块列表 | 日常单据确认，除非公司制度要求 |
| 管理员 | 用户、角色、权限、备份、健康检查 | 系统管理、基础资料 | 代替业务人员做日常单据 |

每天使用 ERP 的推荐节奏：

1. 上班后先登录，看自己岗位相关的待处理列表。
2. 处理当天新增单据。
3. 涉及确认、作废、收付款、库存调整时，认真填写当前密码和操作原因。
4. 下班前核对自己负责的单据是否还有草稿、待确认、异常状态。
5. 发现数据不对时，先查单据和流水，不要让管理员直接改数据库。

### 1.8 单据状态怎么理解

系统里很多单据都有状态。不同模块名称略有差异，但日常理解可以按下面来：

| 状态 | 含义 | 谁处理 |
| --- | --- | --- |
| 草稿 | 刚保存，还没有正式生效 | 单据创建人 |
| 待确认 | 内容已填好，等负责人确认 | 对应岗位负责人 |
| 已确认 | 已正式生效，会影响库存、应收应付或生产进度 | 系统自动记录 |
| 部分完成 | 已做了一部分，例如部分到货、部分领料、部分收款 | 对应岗位继续跟进 |
| 已完成 | 业务闭环完成 | 只查看，不再改 |
| 已作废 | 单据取消，不再参与业务 | 需要权限和操作原因 |

看到异常状态时，先看单据详情里的备注、操作记录和相关流水，再决定由哪个岗位处理。

## 2. 基础资料维护人员

基础资料是后续销售、采购、库存、生产、财务能否顺利使用的前提。

建议基础资料维护顺序：

```text
物料 -> 客户 -> 客户地址 -> 供应商 -> 库位 -> 单位换算 -> 产品组成清单
```

### 2.1 建立物料

菜单位置：

```text
基础资料 -> 物料 -> 新建
```

物料包括成品、原料、配件、包装材料等。

填写字段：

| 字段 | 填写说明 | 示例 |
| --- | --- | --- |
| 物料编码 | 唯一编号，建议字母数字，不要重复 | FG001、RM001、PK001 |
| 物料名称 | 实际名称 | 蓝色成品A、塑料粒子、纸箱 |
| 物料类型 | 按实际选择 | 成品、原料、配件、包装材料 |
| 规格 | 尺寸、型号、颜色 | 20mm 蓝色 |
| 基本单位 | 库存核算单位 | 个、kg、米、张 |
| 数量精度 | 小数位数 | 成品一般 0，重量可填 2 或 3 |
| 最低库存 | 低于多少提醒 | 100 |
| 最新采购价 | 可以先不填，进货后可回写 | 3.50 |
| 状态 | 正常使用选启用 | 启用 |
| 备注 | 补充说明 | 特殊材质 |

操作步骤：

1. 点击“新建”。
2. 填写物料编码和物料名称。
3. 选择正确的物料类型。
4. 填写基本单位。
5. 保存。
6. 回到物料列表，搜索物料编码，确认可以查到。

检查标准：

- 成品必须建为“成品”。
- BOM 里用到的原料、配件、包装材料必须提前建立。
- 库存单位要统一，后面期初库存、采购入库、生产领料都以这个单位为基础。
- 最低库存不能填负数。
- 最新采购价不能填负数；暂时不知道价格时可以留空。

### 2.2 建立客户

菜单位置：

```text
基础资料 -> 客户 -> 新建
```

填写字段：

| 字段 | 填写说明 |
| --- | --- |
| 客户编号 | 客户唯一编号，例如 C001 |
| 客户名称 | 客户完整名称 |
| 简称 | 常用简称 |
| 负责销售 | 负责这个客户的销售人员 |
| 结算方式 | 下拉选择，例如现结、月结30天、款到发货 |
| 联系电话 | 客户常用电话 |
| 状态 | 启用 |
| 备注 | 其他说明 |

操作步骤：

1. 进入客户新建页。
2. 填写客户编号和客户名称。
3. 选择负责销售。
4. 选择结算方式。
5. 保存。
6. 进入客户详情页，继续维护客户地址。

检查标准：

- 普通销售只能看到自己负责客户或自己创建的数据。
- 客户电话属于敏感信息，没有联系信息权限的账号可能看不到。

### 2.3 建立客户地址

菜单位置：

```text
基础资料 -> 客户 -> 打开客户详情 -> 新增地址
```

填写字段：

| 字段 | 填写说明 |
| --- | --- |
| 地址类型 | 收货地址、退货地址、样品地址 |
| 收件人 | 对方收货联系人 |
| 收件电话 | 联系电话 |
| 详细地址 | 完整收货地址 |
| 默认地址 | 常用地址勾选默认 |
| 状态 | 启用 |

建议：

- 常用收货地址设为默认地址。
- 送货单打印会带出客户地址、联系人、电话，建档时要认真核对。

### 2.4 建立供应商

菜单位置：

```text
基础资料 -> 供应商 -> 新建
```

填写字段：

| 字段 | 填写说明 |
| --- | --- |
| 供应商编号 | 供应商唯一编号，例如 S001 |
| 供应商名称 | 供应商完整名称 |
| 联系人 | 对方联系人 |
| 联系电话 | 联系电话 |
| 供应商类型 | 下拉选择，例如原料、辅料、配件、包装、外协加工、运输、设备、服务 |
| 付款方式 | 下拉选择，例如转账、月结30天、预付、货到付款 |
| 状态 | 启用 |
| 备注 | 其他说明 |

检查标准：

- 采购单和供应商付款都需要先有供应商。
- 没有联系信息权限时，部分电话信息可能不显示。

### 2.5 建立库位

菜单位置：

```text
库存与财务 -> 库位 -> 新建
```

填写建议：

| 字段 | 示例 |
| --- | --- |
| 库位编码 | A01、A02、B01 |
| 库位名称 | 原料仓 A01、成品仓 B01 |
| 状态 | 启用 |

建议：

- 上线前至少建立一个默认库位。
- 实物仓库有多个货架时，库位编码尽量和现场标签一致。

### 2.6 维护物料供应商价格

菜单位置：

```text
基础资料 -> 物料 -> 打开物料详情 -> 新增供应商价格
```

这个价格用于采购参考价和后续成本核对。只有有金额权限的账号才能维护或查看。

填写字段：

| 字段 | 填写说明 |
| --- | --- |
| 供应商 | 选择已启用供应商 |
| 采购价 | 当前供应商给这个物料的采购价 |
| 币种 | 默认 CNY |
| 生效日期 | 价格从哪天开始有效 |
| 失效日期 | 价格到哪天结束，不确定可留空 |
| 是否默认 | 常用供应商价格可勾选 |
| 状态 | 启用 |

检查标准：

- 采购价不能填负数。
- 失效日期不能早于生效日期。
- 同一物料只能有一个默认启用供应商价格。
- 价格暂时不确定时，可以先不维护供应商价格，在采购单里按实际填写。

## 3. 产品和产品组成清单维护人员

系统里的产品建议这样理解：

```text
成品物料 = 公司内部产品
产品组成清单 = 这个产品由哪些原料、配件、包装材料组成
客户产品 = 某个客户下单时使用的产品名称，可关联到公司内部成品物料
```

### 3.1 建立一个产品

菜单位置：

```text
基础资料 -> 物料 -> 新建
```

填写方法：

| 字段 | 怎么填 |
| --- | --- |
| 物料编码 | 填成品编码，例如 FG001 |
| 物料名称 | 填产品名称，例如 蓝色包装成品A |
| 物料类型 | 必须选择“成品” |
| 规格 | 填产品规格 |
| 基本单位 | 通常填“个” |
| 数量精度 | 通常填 0 |
| 状态 | 启用 |

保存后，这个成品就可以用于产品组成清单和销售订单。

客户下单时如果使用自己的产品编号或名称，需要在客户详情页新增客户产品。

菜单位置：

```text
基础资料 -> 客户 -> 客户详情 -> 新增客户产品
```

填写方法：

| 字段 | 怎么填 |
| --- | --- |
| 客户产品编号 | 填客户自己的料号、图号或订单型号 |
| 客户产品名称 | 填客户自己的产品名称 |
| 生产成品 | 选择公司内部已经建立的成品物料 |
| 默认售价 | 有金额权限时填写 |
| 标签要求 | 直接填普通文字，例如 无要求、贴客户条码 |
| 包装要求 | 直接填普通文字，例如 塑料包装、纸箱包装 |
| 状态 | 启用 |

注意：

- 标签要求和包装要求不需要写 JSON。
- 如果客户没有自己的编号，可以先填公司内部成品编码。
- 销售订单明细选择的是客户产品，系统会通过客户产品找到对应的生产成品和 BOM。
- 生产成品必须选择，不能留空。
- 默认售价不能填负数；暂时不确定售价时可以留空。

### 3.2 建立产品所需原料和配件

菜单位置：

```text
基础资料 -> 物料 -> 新建
```

把产品里用到的每一种东西都建成物料：

| 组成内容 | 物料类型示例 |
| --- | --- |
| 塑料粒子、布料、胶水 | 原料 |
| 五金扣、螺丝、弹簧 | 配件 |
| 纸箱、标签、包装袋 | 包装材料 |

注意：

- BOM 只能选择已经存在的物料。
- 不要把原料错建成成品。

### 3.3 建立单位换算

什么时候需要单位换算：

| 情况 | 是否需要 |
| --- | --- |
| 库存单位和 BOM 单位一样 | 不需要 |
| 库存用 kg，BOM 用 g | 需要 |
| 库存用 米，BOM 用 cm | 需要 |
| 采购用 箱，库存用 个 | 需要 |

示例：

```text
库存基础单位：kg
BOM 用量单位：g
换算关系：1 g = 0.001 kg
```

建议：

- 能统一单位时尽量统一。
- 不确定换算比例时，先不要启用产品组成清单。
- 源单位和目标单位不能相同。
- 换算比例必须大于 0，不能填 0 或负数。

### 3.4 新建产品组成清单

菜单位置：

```text
基础资料 -> 产品组成清单 -> 新建
```

填写字段：

| 字段 | 怎么填 | 示例 |
| --- | --- | --- |
| 清单编号 | 自己定义，不能重复 | BOM-FG001-A |
| 成品 | 选择刚才建立的成品物料 | FG001 蓝色包装成品A |
| BOM 版本 | 初版通常填 A | A |
| 基准数量 | 通常填 1 | 1 |
| 生效日期 | 点击日历选择从哪天开始使用 | 2026-07-04 |
| 失效日期 | 点击日历选择停用日期，暂时不用可不填 | 空 |
| 备注 | 工艺、版本说明 | 首次建立 |

操作步骤：

1. 点击“新建”。
2. 选择成品。
3. 填清单编号和版本。
4. 基准数量通常填 1。
5. 用日历选择生效日期；暂时没有失效日期时可以留空。
6. 保存。

不同 Windows 版本或浏览器显示的日期样式可能不一样，但系统保存时统一按 `YYYY-MM-DD`。如果电脑没有弹出日历，也可以手工输入 `2026-07-04`、`2026/07/04` 或 `2026.07.04`。

注意：

- 基准数量必须大于 0。
- 失效日期不能早于生效日期。
- 成品下拉框只显示启用状态的成品物料；如果找不到成品，先检查物料是否建成“成品”且状态为“启用”。

### 3.5 添加组成明细

菜单位置：

```text
产品组成清单详情 -> 新增原料 / 配件
```

填写字段：

| 字段 | 怎么填 |
| --- | --- |
| 行号 | 10、20、30 这样填，方便以后插入 |
| 原料 / 配件 | 选择组成物料 |
| 每基准数量用量 | 做 1 个成品要用多少 |
| 单位 | 与用量对应，例如 个、kg、g、米 |
| 损耗率 | 无损耗填 0，5% 填 0.05 |
| 必需物料 | 必须用就勾选 |
| 备注 | 特殊说明 |
| 新增原因 | 例如 新建产品组成 |

示例：

| 行号 | 原料 / 配件 | 每基准数量用量 | 单位 | 损耗率 | 必需 |
| --- | --- | --- | --- | --- | --- |
| 10 | 塑料粒子 | 0.05 | kg | 0.03 | 是 |
| 20 | 金属扣 | 2 | 个 | 0 | 是 |
| 30 | 纸箱 | 1 | 个 | 0 | 是 |
| 40 | 标签纸 | 1 | 张 | 0 | 是 |

### 3.6 启用产品组成清单

产品组成清单建好明细后，需要启用。

操作步骤：

1. 打开产品组成清单详情。
2. 确认明细完整。
3. 在顶部输入当前密码。
4. 填写启用原因，例如“首次建立产品组成清单”。
5. 点击“启用为默认清单”。

启用后：

- 销售订单确认时可以使用这个清单做库存检查。
- 生产指令可以按这个清单生成领料。
- 同一个成品通常只保留一个默认启用清单。

### 3.7 修改已经启用的清单

已经启用的产品组成清单不要直接改。

正确做法：

1. 打开旧清单。
2. 点击“复制为新版本”。
3. 填新清单编号，例如 `BOM-FG001-B`。
4. 填新版本，例如 `B`。
5. 填复制原因。
6. 在新版本上修改明细。
7. 启用新版本为默认清单。

这样历史订单仍能追溯旧版本，新订单使用新版本。

## 12. 导入和导出

### 12.1 导入

适合批量资料：

- 物料。
- 客户。
- 供应商。
- 库位。
- 销售订单。
- 采购单。
- 期初库存。

操作建议：

1. 先下载系统模板。
2. 不要改表头。
3. 用中文表头模板填写。
4. 先导入少量测试数据。
5. 确认无误后再导入正式数据。

注意：

- 旧英文模板不再兼容。
- 中文、英文、数字混合填写不会乱。
- 如果导入失败，先看失败明细，不要反复重复导入。

### 12.2 导出

导出时注意：

- 没有金额权限的账号，导出的金额也会脱敏。
- 导出文件用于核对，不建议再反向手工改数据库。

## 13. 常见问题

### 13.1 为什么看不到某个菜单

原因通常是没有对应权限。

处理：

1. 找管理员。
2. 检查用户角色。
3. 检查角色权限。
4. 修改后重新登录。

### 13.2 为什么金额显示星号

说明当前账号没有“查看财务金额”权限。

这是正常保护，不是系统错误。

### 13.3 为什么确认时要输入当前密码

确认、作废、收付款、库存调整、启用产品组成清单等属于高风险操作。

系统要求再次输入当前密码，用于确认是本人操作。

### 13.4 为什么产品不能下单或不能确认

常见原因：

| 原因 | 处理 |
| --- | --- |
| 成品物料未建立 | 先建成品物料 |
| 产品组成清单未建立 | 先建产品组成清单 |
| 产品组成清单未启用 | 输入密码启用默认清单 |
| 单位换算缺失 | 补单位换算 |
| 库存不足 | 等采购入库或调整计划 |

### 13.5 为什么库存数量不对

排查顺序：

1. 查库存汇总。
2. 查库存批次。
3. 查库存流水。
4. 对照采购入库、生产领料、生产入库、销售出库、退货、盘点调整。
5. 不要直接改数据库。

## 14. 日常完整业务流程

推荐业务闭环：

```text
基础资料建档
-> 建成品和原料
-> 建产品组成清单并启用
-> 建客户和客户地址
-> 销售下单
-> 系统检查库存和欠料
-> 采购处理欠料
-> 进货入库
-> 生产指令
-> 生产领料
-> 生产入库
-> 销售出库和送货单
-> 财务收款核销
-> 老板看经营看板
-> 管理员检查备份和健康状态
```

上线初期建议每天结束前核对：

1. 当日销售订单。
2. 当日进货入库。
3. 当日生产领料和入库。
4. 当日销售出库。
5. 当日收付款。
6. 库存异常。
7. 备份是否成功。
