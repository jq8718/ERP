# 核心页面低保真原型

本文档是 ERP 第一阶段页面开发附件，用文字线框图描述核心页面结构、主要操作、提示信息和用户工作流。

目标：

- 让开发人员知道每个页面第一版应该长什么样。
- 让业务人员能提前确认操作路径。
- 避免进入编码后反复调整入口和按钮。

约定：

- 第一阶段以桌面浏览器为主，审批页面兼顾手机浏览器。
- 页面不做营销式首页，登录后直接进入工作台。
- 所有按钮只是前端入口，后端必须再次校验权限、状态和业务条件。
- 表格列表默认支持搜索、筛选、分页、状态标签和无权限提示。

## 1. 全局布局

```text
┌──────────────────────────────────────────────────────────────┐
│ 顶部栏：公司名 / 当前用户 / 未读消息 / 待审批 / 退出          │
├───────────────┬──────────────────────────────────────────────┤
│ 左侧菜单       │ 主内容区                                      │
│ - 工作台       │                                              │
│ - 基础资料     │                                              │
│ - 销售         │                                              │
│ - 采购         │                                              │
│ - 库存         │                                              │
│ - 生产         │                                              │
│ - 财务         │                                              │
│ - 审批         │                                              │
│ - 系统         │                                              │
└───────────────┴──────────────────────────────────────────────┘
```

全局规则：

- 当前页面标题显示在主内容区顶部。
- 页面右上角放主要动作，例如“新增”“导出”“打印”。
- 操作成功后使用顶部消息条提示。
- 操作失败时显示错误码、中文原因和下一步建议。
- 无权限动作隐藏或置灰；尝试直接访问时显示明确无权限提示。

## 2. 工作台

### 2.1 页面目标

用户登录后第一眼看到自己要处理的事情。

### 2.2 线框

```text
┌ 工作台 ──────────────────────────────────────────────────────┐
│ [我的待审批 12] [欠料预警 5] [借样逾期 3] [待发货 8] [待入库 4] │
├──────────────────────────────┬───────────────────────────────┤
│ 待我审批                     │ 紧急提醒                       │
│ - 销售订单 SO... 金额...     │ - 欠料影响交期                 │
│ - 采购单 PO... 金额...       │ - 借样已逾期 5 天              │
│ [查看全部审批]               │ - 备份失败                     │
├──────────────────────────────┼───────────────────────────────┤
│ 我的待处理任务               │ 今日概览                       │
│ - 待补 BOM                   │ - 新增销售订单                 │
│ - 待确认收款                 │ - 今日出库                     │
│ - 待处理盘点差异             │ - 今日入库                     │
└──────────────────────────────┴───────────────────────────────┘
```

### 2.3 关键交互

- 点击卡片进入对应筛选后的列表。
- 紧急提醒按级别高亮。
- 不同角色看到不同模块数据。
- 无数据时显示“暂无待处理事项”，不显示空白页。

## 3. 基础资料页面

### 3.1 物料列表

```text
┌ 物料管理 ────────────────────────────────────────────────────┐
│ 搜索：[编码/名称] 类型：[全部 v] 状态：[启用 v] [查询] [重置] │
│ [新增物料] [建议编码] [导入] [导出]                            │
├──────────────────────────────────────────────────────────────┤
│ 编码 | 名称 | 类型 | 单位 | 数量精度 | 默认库位 | 状态 | 操作    │
│ RM001| ...  | 原料 | kg   | 3        | A01      | 启用 | 详情 编辑│
└──────────────────────────────────────────────────────────────┘
```

关键交互：

- 新增时物料编码允许手工输入字母数字。
- 系统可给出建议编码，但用户可修改。
- 成品保存时提示“成品后续需要建立 BOM 才能提交销售订单审核”。

### 3.2 客户详情

```text
┌ 客户详情：客户 A ────────────────────────────────────────────┐
│ 基本信息 [编辑] [停用]                                       │
├──────────────────────────────────────────────────────────────┤
│ Tab: 客户产品 | 收件地址 | 销售订单 | 对账/收款 | 附件          │
├──────────────────────────────────────────────────────────────┤
│ 客户产品列表                                                  │
│ 产品编号 | 产品名 | 关联成品 | 默认价 | 标签/包装 | 状态 | 操作 │
│ CP001    | ...    | FG001    | 12.00 | 查看      | 启用 | 编辑 │
└──────────────────────────────────────────────────────────────┘
```

关键交互：

- 客户产品未关联成品时显示醒目标记。
- 销售下单时可从弹窗快速新增客户产品或收件地址。

### 3.3 BOM 详情

```text
┌ BOM 详情：FG001 / V1 ────────────────────────────────────────┐
│ 状态：已启用  [复制为新版本] [停用] [版本对比]                 │
├──────────────────────────────────────────────────────────────┤
│ 成品：FG001  版本：V1  创建人：...                             │
├──────────────────────────────────────────────────────────────┤
│ 子件编码 | 名称 | 用量 | 单位 | 损耗率 | 必需 | 备注             │
│ RM001    | ...  | 0.25 | kg   | 2%     | 是   |                  │
└──────────────────────────────────────────────────────────────┘
```

关键交互：

- 已启用 BOM 不能直接编辑，必须复制为新版本。
- 提交审核前校验子件、单位换算和用量。

```text
┌ 销售订单 SO202606080001 ─────────────────────────────────────┐
│ 状态：待审核  客户：客户 A  交期：2026-06-20                  │

├ 明细行 ─────────────────────────────────────────────────────┤
│ 产品 | 成品编码 | 数量 | 单价 | 金额 | BOM版本 | 库存状态 | 操作 │
│ CP01 | FG001    | 100  | ...  | ...  | V1      | 欠料     | 查看 │
├ 缺失项 / 风险提示 ───────────────────────────────────────────┤
│ - 第 2 行缺少成品编码 [去补充]                                │
│ - FG002 未启用 BOM [去建立 BOM]                                │
├ 相关单据 ───────────────────────────────────────────────────┤
│ 生产指令 / 销售出库 / 收款 / 退货 / 审批记录 / 变更记录         │
└──────────────────────────────────────────────────────────────┘
```

关键交互：

- 草稿可保存缺失项，提交审核前强校验。
- 库存检查结果按明细行展示。
- BOM 版本锁定后在明细行展示。
- 已审核后的关键字段变更走变更记录或审批。

## 11. 页面开发优先级

第一批必须出低保真确认的页面：

1. 工作台。
2. 销售订单列表 / 详情 / 新建编辑。
3. 物料、客户、客户产品、BOM 详情。
4. 欠料列表。
5. 采购单 / 进货单详情。
6. 库存查询、生产领料、销售出库。
7. 收款 / 付款详情。
8. 审批待办 / 审批详情。

第二批页面：

1. 借样单 / 借样归还。
2. 盘点单。
3. 待处理余额。
4. 后台任务 / 健康检查。
5. 导入导出 / 打印记录。

## 12. 原型确认清单

每个页面开工前至少确认：

- 页面入口在哪里。
- 谁有权限看到。
- 默认列表列有哪些。
- 常用筛选有哪些。
- 主按钮有哪些。
- 每个按钮对应哪个服务方法。
- 成功后跳转哪里。
- 失败时提示什么。
- 是否需要二次确认或二次验证。
- 是否需要打印、导出或附件。
